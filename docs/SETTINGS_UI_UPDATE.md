# 设置界面优化说明

## 更新内容

### 1. ✨ 优化了"使用模拟盘"组件布局
- 将 checkbox 改为更美观的 label 包裹形式
- 与"测试API连接"按钮放在同一行，布局更紧凑
- 添加了专门的 `.checkbox-label` 样式，提升视觉效果
- 使用 accent-color 统一主题色

### 2. 🔍 新增"测试API连接"功能
**位置**: OKX API 配置区域，"使用模拟盘"右侧

**功能**:
- 点击按钮后验证当前填写的 API 密钥是否有效
- 通过设置的代理调用 OKX API 获取账户余额
- 实时显示测试结果（成功/失败）

**测试流程**:
1. 填写 API Key、API Secret、API Passphrase
2. 选择是否使用模拟盘
3. （可选）填写代理地址
4. 点击"🔍 测试 API 连接"按钮
5. 等待测试结果

**测试结果**:
- ✅ **成功**: 显示账户余额和环境信息
- ❌ **失败**: 显示具体错误原因
  - API 密钥无效或权限不足
  - IP 地址未加入白名单
  - 连接超时（网络或代理问题）
  - 连接被拒绝（代理配置错误）

## 技术实现

### 前端
**文件**: `public/index.html`
- 重构 OKX API 配置区域 HTML 结构
- 添加注册链接和测试按钮
- 添加测试结果显示区域

**文件**: `public/monitor-styles.css`
- `.section-header` - 区域标题与链接的 flex 布局
- `.register-link` - 注册链接样式
- `.checkbox-label` - 美化的 checkbox 样式
- `.btn-test` - 测试按钮样式
- `.api-test-result` - 测试结果显示区域（支持成功/失败/加载状态）

**文件**: `public/monitor-script.js`
- `testOkxApi()` 方法 - 测试 API 连接逻辑
- 表单验证（检查必填字段）
- 调用后端 `/api/test-okx` 接口
- 动态显示测试结果

### 后端
**文件**: `src/api/routes.ts`
- 新增 `POST /api/test-okx` 端点
- 接收前端传来的 API 凭证
- 创建临时 OKX 客户端实例
- 调用 `getFuturesAccount()` 获取账户信息
- 返回余额或友好的错误信息

**错误处理**:
- 401/Unauthorized → "API 密钥无效或权限不足"
- 403/Forbidden → "IP 地址未加入白名单"
- timeout → "连接超时，请检查网络或代理设置"
- ECONNREFUSED → "连接被拒绝，请检查代理设置"

## 注意事项

1. **API 密钥安全**: 测试时使用的是临时客户端实例，不会影响全局配置
2. **代理设置**: 如果配置了代理，测试时会使用代理连接
3. **模拟盘/正式盘**: 确保选择正确的环境进行测试
4. **IP 白名单**: 确保在 OKX 官网添加了服务器 IP 或代理 IP 到白名单
