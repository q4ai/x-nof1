# 配置管理系统说明

## 概述

系统现已支持通过 Web 界面动态管理所有配置，包括敏感的 API 密钥信息。所有配置（除 `PORT` 和 `DATABASE_URL` 外）都存储在数据库中，可以在运行时修改并重新加载。

## 访问设置界面

1. 打开监控面板：`http://localhost:3100`
2. 点击右上角的 **⚙️ 设置按钮**
3. 在弹出的对话框中修改配置

## 配置分类

### 🔑 OKX API 配置
- **API Key** - OKX API 密钥（必填）
- **API Secret** - OKX API 私钥（必填）
- **API Passphrase** - OKX 交易口令（必填）
- **使用模拟盘** - 勾选后使用测试环境

### 🤖 AI 模型配置
- **OpenAI API Key** - AI 服务 API 密钥（必填）
- **API 基础地址** - 如 OpenRouter、DeepSeek 等
- **AI 模型名称** - 具体使用的模型

### 💼 交易配置
- **交易币种** - 逗号分隔的币种列表
- **交易间隔** - 决策周期（分钟）
- **交易策略** - 超短线/波段趋势/稳健/平衡/激进

### ⚠️ 风险参数
- **最大杠杆倍数**
- **最大持仓数**
- **最大持仓时间**
- **极端止损百分比**

### 💰 账户参数
- **初始资金**
- **止损线**
- **止盈线**

### 📉 回撤风控
- **警告阈值**
- **禁止开仓阈值**
- **强制平仓阈值**

### 🌐 网络配置
- **HTTP 代理** - 可选的代理地址

## 安全特性

### 敏感信息保护
- API 密钥在界面显示时自动脱敏（如：`17a1***e6d`）
- 支持密码字段的显示/隐藏切换（点击 👁️ 图标）
- 只有修改后的密钥才会被保存，未修改的保持原值

### 传输安全
- 敏感字段仅在用户主动修改时才发送到服务器
- 包含 `***` 的脱敏值不会被保存

## 使用流程

### 首次启动
1. 系统会从 `.env` 文件读取配置并初始化到数据库
2. 后续可以通过设置界面管理，无需再编辑 `.env`

### 修改配置
1. 打开设置对话框
2. 修改需要更改的字段
3. 点击 **保存并重启** 按钮
4. 系统会自动重新加载配置并重置交易客户端

### 配置重载机制
保存配置后会触发：
- 数据库配置更新
- 风险参数重新加载
- OKX 客户端实例重置（使用新的 API 密钥）
- 前端页面刷新

## 技术实现

### 后端
- `src/database/init-config.ts` - 配置初始化和读写
- `src/config/riskParams.new.ts` - 动态配置加载
- `src/services/okxClient.ts` - 客户端重置机制
- `src/api/routes.ts` - 配置 API 端点

### 前端
- `public/index.html` - 设置界面 UI
- `public/monitor-script.js` - 配置加载和保存逻辑
- `public/monitor-styles.css` - 样式定义

### API 端点
- `GET /api/config` - 获取配置（脱敏版本）
- `PUT /api/config` - 更新配置
- `POST /api/reload` - 重新加载配置

## 注意事项

1. **PORT 和 DATABASE_URL** 仍需在 `.env` 中配置（不可通过界面修改）
2. 修改配置后建议重启服务以确保所有模块都使用新配置
3. API 密钥修改后会立即生效，但交易循环可能需要等待下个周期
4. 请妥善保管 API 密钥，虽然界面已脱敏，但数据库中存储的是明文

## 故障排查

### 配置保存失败
- 检查数据库连接是否正常
- 查看浏览器控制台错误信息
- 检查后端日志

### 配置重载后不生效
- 手动重启服务：`npm run dev`
- 或使用 PM2：`pm2 restart ecosystem.config.cjs`

### API 密钥无效
- 确认在 OKX 官网设置的 IP 白名单
- 检查是否选择了正确的环境（正式/模拟盘）
- 验证 Passphrase 是否正确
